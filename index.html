sistemas operacionais

  • Um sistema operacional (SO) é um software fundamental que atua como uma interface entre o hardware do computador e os aplicativos de software.
  • Ele gerencia recursos de hardware, fornece serviços para aplicativos e oferece uma interface para os usuários interagirem com o computador.

  • Definição do modelo de logins

    • 1. Identificação do Usuário:

  • Estabelecer como os usuários serão identificados no sistema, seja por meio de nomes de usuário, endereços de e-mail ou outros identificadores exclusivos.

    • 2. Autenticação:

  • Definir os métodos de autenticação utilizados para verificar a identidade dos usuários. Isso pode incluir senhas, autenticação de dois fatores (2FA), autenticação biométrica, entre outros.

    • 3. Políticas de Senha:

  • Estabelecer políticas relacionadas às senhas, como requisitos de complexidade, expiração, e regras para troca periódica.

    • 4. Bloqueio de Conta e Tentativas Malsucedidas:

  • Implementar medidas de segurança, como bloqueio de contas após um número específico de tentativas malsucedidas, para proteger contra ataques de força bruta.

    • 5. Registro de Atividades (Logs):

  • Configurar a geração de logs para registrar atividades relacionadas aos logins, facilitando a auditoria e detecção de atividades suspeitas.

    • 6. Segurança da Transmissão:

  • Garantir que as informações de login sejam transmitidas de forma segura por meio de criptografia, especialmente em ambientes online.

    • 7. Gestão de Sessões:

  • Estabelecer políticas para gerenciar sessões de usuários ativos, incluindo tempo limite de inatividade e ações permitidas durante uma sessão.

    • 8. Recuperação de Conta:

  • Implementar procedimentos seguros para a recuperação de contas, como perguntas de segurança ou links de recuperação enviados por e-mail.

    • 9. Integração com Sistemas de Terceiros:

  • Se necessário, considerar como o sistema de login se integra com sistemas de autenticação de terceiros, como redes sociais ou serviços de identidade federada.

    • 10. Conformidade com Regulamentações:

  • Verificar se o modelo de logins está em conformidade com regulamentações de privacidade e segurança, como o GDPR (Regulamento Geral de Proteção de Dados) ou normas específicas do setor.

    • 11. Notificações de Atividade Suspeita:

  • Implementar notificações automáticas para usuários ou administradores quando atividades de login suspeitas são detectadas.

    • Instalação do SO a partir de periféricos.

    1. Escolha do Sistema Operacional:

  • Escolha o sistema operacional que deseja instalar e faça o download da imagem de instalação correspondente. Pode ser o Windows, Linux, macOS, etc.

    • 2. Mídia de Instalação Inicializável:

  • Conecte um pendrive USB ao seu computador.
  • Formate o pendrive para garantir que esteja vazio e pronto para uso.
  • Use um software de criação de mídia de inicialização, como o Rufus (para Windows) ou o balenaEtcher (multiplataforma), para gravar a imagem do SO no pendrive.

    • 3. Configuração da BIOS/UEFI:

  • Reinicie ou ligue o computador.
  • Pressione a tecla apropriada para acessar o menu de configuração da BIOS/UEFI. Isso geralmente é feito pressionando uma tecla como F2, F10, Del ou Esc durante a inicialização.
  • Vá para a seção de configuração de inicialização e altere a ordem de inicialização para que o periférico a partir do qual deseja instalar (pendrive ou DVD) esteja no topo da lista.

    • 4. Inicialização e Instalação:

  • Salve as configurações da BIOS/UEFI e saia.
  • O computador será reiniciado a partir do periférico escolhido, e você verá a tela de instalação do sistema operacional.
  • Siga as instruções na tela para iniciar o processo de instalação.
